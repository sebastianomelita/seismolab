---Librerie opensource di sviluppatori terzi adoperate

--ESP8266wifi
A simple ESP8266 Arduino library with built in re-connect functionality.
https://github.com/ekstrand/ESP8266wifi
Particolarità: comandi AT memorizzati su area istruzioni per risparmiare spazio per i dati (tramite direttiva PROGMEM)


--WeeESP8266
An ESP8266 library for Arduino providing an easy-to-use way to manipulate ESP8266.
Source can be download at https://github.com/itead/ITEADLIB_Arduino_WeeESP8266.
Libreria alternativa.
Particolarità: occupa più spazio in memoria RAM, presumibilmente più rapida.


--I2Cdev library collection 
MPU6050 I2C device classBased on InvenSense MPU-6050 register map document rev. 2.0, 5/19/2011 (RM-MPU-6000A-00)
10/3/2011 by Jeff Rowberg <jeff@rowberg.net>
Updates should (hopefully) always be available at
Contiene una libreria molto colpleta per la gestione delmodulo accelerometro MPU6050.
https://github.com/jrowberg/i2cdevlib


--Fishino libraries
Librerie della scheda arduino compatibile con il modulo WIFI ESP8266 integrato on board.
Utilizzata per ottenere riferimenti per creare un wrapper della libreria ESP8266wifi che esponga le funzioni usuali delle librerie del modulo Etherner Shield di Arduino
http://fishino.it/en/download/


-----Prototipi realizzati:
Particolarità: utilizzano l'accelerometro digitale MPU6050, si collegano ad internet tramite il modulo wifi ESP8266, implementano l'algoritmo rilevatore di soglia del prototipo seismocloud basato su raspberry/Galileo.
Le librerie (ampiamente integrate e rimaneggiate rispetto alle originali) di cui abbisognano sono già incluse nelle cartelle dei prototipi.
Le funzioni di rete appaiono grossomodo analoghe a quelle in uso sullo shield ethernet.

--seismocloudArduinoUnoCorto utilizza le librerie ESP8266wifi
Scelto perchè consuma minori risorse in RAM (più adatto per il meno costoso Arduino Uno).
Attualmente completo. Compila ed esegue (apparentemente in maniera corretta) tutte le funzioni pertanto è in fase di collaudo. 
E' dotato di tutte le funzioni dell'originale seismocloud basato su Arduino Uno ma in più ha: la funzione di calibrazione automatica dell'accelerometro in corrispondenza di alcune situazioni (evento terremotro in seguito ad uno spostamento o riposizionamento, evento di http alive presso il server), la soglia di rilevazione terremoti basata sul calcolo della deviazione standard del modello raspberry pi (il valore di scatto è però mediato con un filtro esponenziale), il parametro sigma viene adesso caricato dal server, il MAC del dispositivo è adesso quello del modulo ESP8266, infine si connette al server syslog (stesso indirizzo keep alive) alla porta 514 per la registrazione dei dati dell'allerta (quake).
